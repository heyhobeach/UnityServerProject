해당 프로젝트를 처음 만들게 된 이유

문제 발생
유저의 사망횟수 및 플레이 타임을 수기로 기록하니 유저가 많을때 기록자의 피로도와 기록의 오류가 발생 하고 모든 전시, 테스트가 종료된 이후 해당 기록으로 볼때 각각의 기록자 별로 편차가 존재 했습니다.

해결 방안 제시
편차를 줄이기 위해서 유저의 플레이 타임과 사망횟수 기록을 자동으로 측정 할 수 있으면 좋겠다 라고 판단 하였고 오프라인 전시만 진행 할 경우 로컬파일로 저장해도 되었으나 추후 온라인 전시에서는 로컬파일로 저장할 경우 확인 할 수 없는 문제가 있어 기록용 서버를 제작하여 게임에 도입을 할 경우 해당 문제를 해결 할 수 있다고 판단 했습니다.

성과
현재 프로젝트는 아직 정말 간단한 정보만 기록 하였으나 기획자의 경우 직접 기록하고 유저들의 플레이 타임을 모두 기억 할 수는 없어 플레이 타임 기록에 대한 정보를 유의미 하게 사용하였습니다. 해당 정보를 통해서 난이도 조정, 유저의 실력의 편차를 파악 할 수 있었습니다.

추후 개선 방안
현재 서버는 아주 기초적인 정보만 기록중입니다. 해당 정보는 유저의 플레이 타임 같은 간단한 정보는 기록 할 수 있으나 어떤 스테이지에서 많이 유저가 고전하는지, 미니게임 통과률이 어느정도인지 파악에 대한 문제가 있습니다. 해당 정보들은 추가적으로 기록하고 시각화를 통해 기획자가 조금 더 많은 정보를 얻을 수 있도록 개선 해야 합니다.
